# Railway Production Runtime Guards

These guards apply to the Railway `Pollek-Cloud` production environment.

## Local Control Plane reconcile

```text
POLLEK_LCP_WATCH=disabled
```

The Cloud server's development default is `http://127.0.0.1:43891`. That address
does not represent an LCP inside a Railway container. Leaving the reconcile
watcher enabled without an explicitly reachable relay creates repeated failed
tasks, audit events, and telemetry noise.

Production keeps the watcher disabled while it receives LCP updates through
explicit ingest and change-batch flows. Enable the watcher only when
`POLLEK_LCP_URL` identifies an approved, authenticated, tenant-scoped relay that
is reachable from Railway.

Before enabling the watcher:

1. Register the relay or LCP through the approved enrollment flow.
2. Require the secure-channel controls defined in
   `docs/HANDOFF_RAILWAY_PHASE_B_MTLS_SVID.md`.
3. Verify tenant/device identity and endpoint allowlisting.
4. Set a bounded reconcile interval with jitter.
5. Test one tenant in monitor mode and confirm the task, audit, and event streams
   remain quiet when no data changes.
6. Record rollback ownership and return to `POLLEK_LCP_WATCH=disabled` if the
   relay is unavailable or identity verification fails.

On 2026-07-23, failed reconcile artifacts generated by the localhost default
were removed from the production PostgreSQL snapshot. The production UI was
then verified with an empty fleet and no localhost reconcile failures.
